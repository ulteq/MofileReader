4 An RFC 5322 email address parser.

6 v 3.0.2
7 
8 What?
9 -----
10 Want to see if something could be an email address? Want to grab the display name or just the address out of a string? Put your regexes down and use this parser!
11 
12 This library does not validate email addresses - we can't really do that without sending an email. However, it attempts to parse addresses using the (fairly liberal) grammar specified in RFC 5322. You can use this to check if user input looks like an email address.
13 
14 Note carefully though - this parser supports all features of RFC 5322, which means that `"Bob Example" <bob@example.com>`
15 is a valid email address. If you just want to validate the `bob@example.com` part, that is RFC 5321, for which you want
16 to use something like node-address-rfc2821.
17 
18 Why use this?
19 -------------
20 Use this library because you can be sure it really respects the RFC:
21  - The functions in the recursive decent parser match up with the productions in the RFC
22  - The productions from the RFC are written above each function for easy verification
23  - Tests include all of the test cases from the [is_email](https://github.com/dominicsayers/isemail) project, which are extensive

84 API
85 ---
86 
87 `obj = addrs(opts)`
88 ===================
89 
90 Call the module directly as a function to get access to the AST. Returns null for a failed parse (an invalid
91 address).
92 
93 Options:
94 
95 * `string` - An email address to parse. Parses as `address-list`, a list of email addresses separated by commas.
96 * `object` with the following keys:
97  * `input` - An email address to parse. Required.
98  * `rfc6532` - Enable rfc6532 support (unicode in email addresses). Default: `false`.
99  * `partial` - Allow a failed parse to return the AST it managed to produce so far. Default: `false`.
100  * `simple` - Return just the address or addresses parsed. Default: `false`.
101  * `strict` - Turn off features of RFC 5322 marked "Obsolete". Default: `false`.
102  * `rejectTLD` - Require at least one `.` in domain names. Default: `false`.
103  * `startAt` - Start the parser at one of `address-list`, `from`, `sender`, `reply-to`. Default: `address-list`.
104 
105 Returns an object with the following properties:
106 
107 * `ast` - the full AST of the parse.
108 * `addresses` - array of addresses found. Each has the following properties:
109  * `parts` - components of the AST that make up the address.
110  * `type` - The type of the node, e.g. `mailbox`, `address`, `group`.
111  * `name` - The extracted name from the email. e.g. parsing `"Bob" <bob@example.com>` will give `Bob` for the `name`.
112  * `address` - The full email address. e.g. parsing the above will give `bob@example.com` for the `address`.
113  * `local` - The local part. e.g. parsing the above will give `bob` for `local`.
114  * `domain` - The domain part. e.g. parsing the above will give `example.com` for `domain`.
115 
116 Note if `simple` is set, the return will be an array of addresses rather than the object above.
117 
118 Note that addresses can contain a `group` address, which in contrast to the `address` objects
119 will simply contain two properties: a `name` and `addresses` which is an array of the addresses in
120 the group. You can identify groups because they will have a `type` of `group`. A group looks
121 something like this: `Managing Partners:ben@example.com,carol@example.com;`
122 
123 `obj = addrs.parseOneAddress(opts)`
124 ===================================
125 
126 Parse a single email address.
127 
128 Operates similarly to `addrs(opts)`, with the exception that `rfc6532` and `simple` default to `true`.
129 
130 Returns a single address object as described above. If you set `simple: false` the returned object
131 includes a `node` object that contains the AST for the address.
132 
133 `obj = addrs.parseAddressList(opts)`
134 ====================================
135 
136 Parse a list of email addresses separated by comma.
137 
138 Operates similarly to `addrs(opts)`, with the exception that `rfc6532` and `simple` default to `true`.
139 
140 Returns a list of address objects as described above. If you set `simple: false` each address will
141 include a `node` object that containst the AST for the address.
142 
143 `obj = addrs.parseFrom(opts)`
144 =============================
145 
146 Parse an email header "From:" address (specified as mailbox-list or address-list).
147 
148 Operates similarly to `addrs(opts)`, with the exception that `rfc6532` and `simple` default to `true`.
149 
150 Returns a list of address objects as described above. If you set `simple: false` each address will
151 include a `node` object that containst the AST for the address.
152 
153 `obj = addrs.parseSender(opts)`
154 ===============================
155 
156 Parse an email header "Sender:" address (specified as mailbox or address).
157 
158 Operates similarly to `addrs(opts)`, with the exception that `rfc6532` and `simple` default to `true`.
159 
160 Returns a single address object as described above. If you set `simple: false` the returned object
161 includes a `node` object that contains the AST for the address.
162 
163 `obj = addrs.parseReplyTo(opts)`
164 ================================
165 
166 Parse an email header "Reply-To:" address (specified as address-list).
167 
168 Operates identically to `addrs.parseAddressList(opts)`.

170 Usage
171 -----
172 If you want to simply check whether an address or address list parses, you'll want to call the following functions and check whether the results are null or not: ```parseOneAddress``` for a single address and ```parseAddressList``` for multiple addresses.
173 
174 If you want to examine the parsed address, for example to extract a name or address, you have some options. The object returned by ```parseOneAddress``` has four helper values on it: ```name```, ```address```, ```local```, and ```domain```. See the example above to understand is actually returned. (These are equivalent to ```parts.name.semantic```, ```parts.address.semantic```, etc.) These values try to be smart about collapsing whitespace, quotations, and excluding RFC 5322 comments. If you desire, you can also obtain the raw parsed tokens or semantic tokens for those fields. The ```parts``` value is an object referencing nodes in the AST generated. Nodes in the AST have two values of interest here, ```tokens``` and ```semantic```.
175 
176 ```
177 > a = addrs.parseOneAddress('Jack Bowman <jack@fogcreek.com >')
178 > a.parts.name.tokens
179 'Jack Bowman '
180 > a.name
181 'Jack Bowman'
182 > a.parts.name.semantic
183 'Jack Bowman '
184 > a.parts.address.tokens
185 'jack@fogcreek.com '
186 > a.address
187 'jack@fogcreek.com'
188 > a.parts.address.semantic
189 'jack@fogcreek.com'
190 ```
191 
192 If you need to, you can inspect the AST directly. The entire AST is returned when calling the module's function.
