1 # seek-bzip
2 
3 [![Build Status][1]][2] [![dependency status][3]][4] [![dev dependency status][5]][6]
4 
5 `seek-bzip` is a pure-javascript Node.JS module adapted from [node-bzip](https://github.com/skeggse/node-bzip) and before that [antimatter15's pure-javascript bzip2 decoder](https://github.com/antimatter15/bzip2.js). Like these projects, `seek-bzip` only does decompression (see [compressjs](https://github.com/cscott/compressjs) if you need compression code). Unlike those other projects, `seek-bzip` can seek to and decode single blocks from the bzip2 file.
6 
7 `seek-bzip` primarily decodes buffers into other buffers, synchronously.
8 With the help of the [fibers](https://github.com/laverdet/node-fibers)
9 package, it can operate on node streams; see `test/stream.js` for an
10 example.
11 
12 ## How to Install
13 
14 ```
15 npm install seek-bzip
16 ```
17 
18 This package uses
19 [Typed Arrays](https://developer.mozilla.org/en-US/docs/JavaScript/Typed_arrays), which are present in node.js >= 0.5.5.
20 
21 ## Usage
22 
23 After compressing some example data into `example.bz2`, the following will recreate that original data and save it to `example`:
24 
25 ```
26 var Bunzip = require('seek-bzip');
27 var fs = require('fs');
28 
29 var compressedData = fs.readFileSync('example.bz2');
30 var data = Bunzip.decode(compressedData);
31 
32 fs.writeFileSync('example', data);
33 ```
34 
35 See the tests in the `tests/` directory for further usage examples.
36 
37 For uncompressing single blocks of bzip2-compressed data, you will need
38 an out-of-band index listing the start of each bzip2 block. (Presumably
39 you generate this at the same time as you index the start of the information
40 you wish to seek to inside the compressed file.) The `seek-bzip` module
41 has been designed to be compatible with the C implementation `seek-bzip2`
42 available from https://bitbucket.org/james_taylor/seek-bzip2. That codebase
43 contains a `bzip-table` tool which will generate bzip2 block start indices.
44 There is also a pure-JavaScript `seek-bzip-table` tool in this package's
45 `bin` directory.

47 ## Documentation
48 
49 `require('seek-bzip')` returns a `Bunzip` object. It contains three static
50 methods. The first is a function accepting one or two parameters:
51 
52 `Bunzip.decode = function(input, [Number expectedSize] or [output], [boolean multistream])`
53 
54 The `input` argument can be a "stream" object (which must implement the
55 `readByte` method), or a `Buffer`.
56 
57 If `expectedSize` is not present, `decodeBzip` simply decodes `input` and
58 returns the resulting `Buffer`.
59 
60 If `expectedSize` is present (and numeric), `decodeBzip` will store
61 the results in a `Buffer` of length `expectedSize`, and throw an error
62 in the case that the size of the decoded data does not match
63 `expectedSize`.
64 
65 If you pass a non-numeric second parameter, it can either be a `Buffer`
66 object (which must be of the correct length; an error will be thrown if
67 the size of the decoded data does not match the buffer length) or
68 a "stream" object (which must implement a `writeByte` method).
69 
70 The optional third `multistream` parameter, if true, attempts to continue
71 reading past the end of the bzip2 file. This supports "multistream"
72 bzip2 files, which are simply multiple bzip2 files concatenated together.
73 If this argument is true, the input stream must have an `eof` method
74 which returns true when the end of the input has been reached.
75 
76 The second exported method is a function accepting two or three parameters:
77 
78 `Bunzip.decodeBlock = function(input, Number blockStartBits, [Number expectedSize] or [output])`
79 
80 The `input` and `expectedSize`/`output` parameters are as above.
81 The `blockStartBits` parameter gives the start of the desired block, in bits.
82 
83 If passing a stream as the `input` parameter, it must implement the
84 `seek` method.
85 
86 The final exported method is a function accepting two or three parameters:
87 
88 `Bunzip.table = function(input, Function callback, [boolean multistream])`
89 
90 The `input` and `multistream` parameters are identical to those for the
91 `decode` method.
92 
93 This function will invoke `callback(position, size)` once per bzip2 block,
94 where `position` gives the starting position of the block (in *bits*), and
95 `size` gives the uncompressed size of the block (in bytes).
96 
97 This can be used to construct an index allowing direct access to a particular
98 block inside a bzip2 file, using the `decodeBlock` method.
99 
100 ## Command-line
101 There are binaries available in bin. The first generates an index of all
102 the blocks in a bzip2-compressed file:
103 ```
104 $ bin/seek-bzip-table test/sample4.bz2
105 32 99981
106 320555 99981
107 606348 99981
108 847568 99981
109 1089094 99981
110 1343625 99981
111 1596228 99981
112 1843336 99981
113 2090919 99981
114 2342106 39019
115 $
116 ```
117 The first field is the starting position of the block, in bits, and the
118 second field is the length of the block, in bytes.
119 
120 The second binary decodes an arbitrary block of a bzip2 file:
121 ```
122 $ bin/seek-bunzip -d -b 2342106 test/sample4.bz2 | tail
123 élan's
124 émigré
125 émigré's
126 émigrés
127 épée
128 épée's
129 épées
130 étude
131 étude's
132 études
133 $
134 ```
135 
136 Use `--help` to see other options.
